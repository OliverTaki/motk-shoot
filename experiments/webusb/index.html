WebUSB/PTP camera lab

Try libgphoto2 in the browser without the camera agent. This lab is isolated from normal projects and never runs unless you open it.

Do not use this for an irreplaceable take. The upstream Web-gPhoto2 package is archived and camera/driver support varies. The regular tether agent remains the production path.

Known boundaries

  • Chromium-based browser, HTTPS/localhost, and cross-origin isolation are required.
  • Windows normally needs the camera interface assigned to WinUSB (for example with Zadig); this can stop native camera software from seeing it until the driver is restored.
  • Only the separate lab server supplies the required COOP/COEP headers.
  • No WebUSB result is written into MOTK Shoot projects. The agent workflow is unchanged.
